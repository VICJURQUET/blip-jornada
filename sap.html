Land — Venda para Novos Clientes (Novo CNPJ)
Este fluxo se aplica sempre que houver uma venda para um CNPJ que ainda não existe no SAP. O vendedor é o primeiro elo da cadeia: o que for preenchido aqui é exatamente o que o cliente será cobrado. Atenção total ao preenchimento.
⚡
Faça este passo a passo ANTES do ganho!
⚠️
Atenção, time de Expand: se o cliente realizou uma aquisição com um CNPJ que não possui contrato ativo no SAP, o processo de Land deve ser seguido integralmente — ou seja, o Manual de Atividades de Novos Clientes [SAP] deve ser preenchido e o item de linha deve referenciar a nova compra.
Caso esse processo seja conduzido dentro de uma pipeline de Expand, preencha o campo "ID de Plataforma" com 00000.
1
Adicionar e Configurar o Item de Linha
Etapas: Negociação e OK Formal
▾
📌 O que fazer
Acesse o Item de Linha do deal
Dentro do deal, acesse a seção de itens de linha. Para editar os campos SAP, clique em Ações → Editar no item desejado. Customize apenas os campos que sofreram alteração na negociação — campos em branco devem permanecer em branco.
🚫 Prazo precisa ser diferente de zero, o valor mínimo é 1. Qualquer item com prazo zerado será rejeitado pela integração.
⚠️ Vendas personalizadas. Para contratos fora do padrão (desconto ou prazo diferenciado), a personalização é obrigatória e deve ser feita com muito cuidado. Em caso de dúvida, acione sua coordenação antes de avançar!
📸 PRINT-01 — Tela de itens de linha com campos [SAP] e Prazo destacados, pois precisam ser preenchidos manualmente
📸 PRINT-02 — Para personalizar o item de linha vá em Ações → Editar
⚠️ Atenção: Todos os campos que já vêm automaticamente preenchidos são campos que refletem no SAP! Tome muito cuidado na edição do item — sempre garanta um double check!
⚠️ Sempre que houver uma negociação no Item de Linha, os campos abaixo devem ser revisados. Quando a negociação fugir do padrão, é obrigatório revisitar e personalizar manualmente, garantindo que reflitam exatamente o que foi vendido e formalizado.

Expand — Nova Venda para Clientes da base (CNPJ com contrato ativo na Blip)
Estes fluxos se aplicam a clientes que já possuem contrato ativo no SAP. O campo [SAP] ID Deal de Plataforma é obrigatório em todos os cenários Expand — ele é o vínculo que conecta a nova venda ao contrato principal do cliente.
⚠
PONTO DE ATENÇÃO — Onde encontrar o ID Deal de Plataforma
Leia com calma antes de avançar o card.
▾
🚨
Essa é a informação mais importante de todo o processo de Expand.
Se o ID Deal de Plataforma estiver errado, a venda não conecta ao contrato do cliente no SAP — e a integração falha.
📌 Seção 1 · Como encontrar o ID correto
Passo a passo para preencher o campo certo
Passo 1 · No HubSpot, abra a empresa do cliente.
Passo 2 · Dentro da empresa, localize a seção Deal de Plataforma e copie o ID do Registro — esse é o número que vai no deal.
📷 Print — Seção "Deal de Plataforma" na empresa
Dentro da empresa no HubSpot, role até a seção Deal de Plataforma e copie o número que aparece na coluna ID DO REGISTRO.
Passo 3 · No deal, clique em Ações e depois em "Exibir todas as propriedades" para localizar o campo [SAP] ID Deal de Plataforma.
📷 Print — Ações › Exibir todas as propriedades
Dentro do deal, no card "Sobre esse Negócio", clique em Ações e selecione "Exibir todas as propriedades".
Passo 4 · No painel "Todas as propriedades", pesquise por "sap", localize o campo [SAP] ID Deal de plataforma e cole o ID do Registro copiado.
📷 Print — Campo [SAP] ID Deal de plataforma
No painel lateral "Todas as propriedades", digite sap na busca e cole o ID do Registro no campo [SAP] ID Deal de plataforma.
⚠️ Sempre escolha o ID do Registro vinculado ao contrato do CNPJ que você está vinculando à expansão.
🚨
E se a empresa tiver mais de um ID de Plataforma?
NÃO escolha por conta própria. Entre em contato com o Analista Financeiro da sua BU para confirmar qual é o ID correto que deve ser vinculado nesta ação. Vincular o ID errado faz a integração com o SAP falhar.
📍 Seção 2 · Onde preencher esse campo no processo
O campo é solicitado em duas pipelines diferentes
O vínculo do deal com o contrato principal acontece quando você preenche o [SAP] ID Deal de Plataforma nos momentos certos das pipelines de [SMB] Expand e Expansão.

Em qual passagem de etapa preencher:

[SMB] Expand

  • Da etapa Mapeamento → Prospecção

Expansão

  • Da etapa Prospecção → OPPs
💡 Sem esse preenchimento nesses momentos, o deal não progride corretamente — ou pior, avança com vínculo errado e quebra a integração com o SAP.
🛠️ Seção 3 · E se eu preenchi o número errado?
Passo a passo para corrigir o ID Deal de Plataforma
Identificou que colocou o número errado? Siga este passo a passo antes que a integração falhe.
Passo 1 · Entre na empresa vinculada ao deal.
Passo 2 · Acesse a visão geral da empresa.
Passo 3 · Copie o ID do Registro da empresa.
Passo 4 · Acesse o deal que está com o ID errado.
Passo 5 · Clique em "Ver todas as propriedades" do deal.
Passo 6 · Cole o ID do Registro correto no campo [SAP] Id de Plataforma.
🚨
Lembre-se: se a empresa tiver mais de um ID de Plataforma, NÃO escolha por conta própria. Acione o Analista Financeiro da sua BU antes de colar qualquer número.

Aqui você só desativa o item específico (um número adicional ou um add-on). Se o cliente quer encerrar o contrato inteiro, é um fluxo de Churn — acione a coordenação.
